# Hyperion Certification Security Assessment Plan

- Kernel version: 0.1.0
- ABI version: 2
- Checked on: 2026-05-23
- Scope: third-party security assessment plan for `CERT-OPEN-008`
- Boundary: this plan does not close `CERT-OPEN-008`; an accepted external assessment report is still required.

## Required Report Metadata
| Field | Requirement |
| --- | --- |
| source_commit | exact git commit and build manifest hash under review |
| submitted_binary_hash | SHA-256 of the assessed kernel binary |
| profile_bundle_hash | SHA-256 of the signed profile and CAPK bundle used during testing |
| target_device_scope | terminal model, firmware, reader interface, and PED boundary assessed |
| assessor_identity | organization, reviewer, report date, and accepted assessment method |
| finding_disposition | critical/high findings remediated or formally accepted with retest evidence |

## Assessment Controls
| ID | Surface | Threat | Repository Evidence | Assessor Evidence Required | Acceptance Gate |
| --- | --- | --- | --- | --- | --- |
| SEC-ASSESS-APDU-INJECTION | APDU replay and command construction | malformed or out-of-sequence APDUs bypass transaction state or parser bounds | trace::tests::replay_rejects_structurally_invalid_command_apdus, trace::tests::replay_rejects_apdu_payloads_above_max_bytes, ffi::tests::transmit_apdu_followups_rejects_chains_above_limit, traceability_foundation::krn_cert_004_penetration_rejects_apdu_injection_and_state_bypass | penetration-test cases for malformed CLA/INS/P1/P2/Lc/Le, chained follow-up loops, and replay-order manipulation | no APDU injection path may skip mandatory state transitions, expose sensitive values, or panic |
| SEC-ASSESS-FSM-BYPASS | transaction finite-state machine | host, card, or callback behavior forces issuer auth, scripts, final GAC, or outcome handling out of order | fsm::tests::rejects_state_machine_annex_schema_and_semantic_drift, fsm::tests::asynchronous_failures_are_explicit_error_transitions, ffi::tests::krn_api_004_rejects_reentrant_mutating_entrypoints, traceability_foundation::rtm_promotes_fsm_annex_replay_and_error_transition_evidence | architecture review and negative tests for illegal callback ordering, reentrancy, timeout, retry, and host-response sequencing | all invalid state transitions must fail closed with stable errors and no partial outcome claim |
| SEC-ASSESS-TRACE-LEAKAGE | trace, debug, and report output | PAN, Track 2, issuer scripts, cryptograms, PIN handles, or recovered key material leak through logs | trace::tests::production_policy_never_emits_full_apdu_data_even_if_misconfigured, trace::tests::production_suppresses_transaction_cryptograms, trace::tests::production_suppresses_issuer_script_command_data, record::tests::summarizes_track2_equivalent_data_without_exposing_pan, oda::tests::oda_debug_redacts_recovered_authentication_material | log-scraping review across success, decline, parser-failure, issuer-script, ODA, and crash-safety paths | production traces and debug output must contain only masked or structural values for sensitive data |
| SEC-ASSESS-PROFILE-TAMPERING | signed profile and CAPK loading | rollback, unsigned profile, expired CAPK, checksum drift, or placeholder data enters certification mode | config::tests::rejects_unsigned_certification_profile_rollback_and_replay, config::tests::rejects_certification_capk_checksum_mismatch_or_metadata_drift, config::tests::rejects_example_profile_in_certification_or_production_mode, traceability_foundation::profile_loader_rejects_rollback_placeholders_and_expired_capks | tamper tests for profile signature status, rollback counter, CAPK checksum, expiry, material status, and RID/AID boundaries | certification and production policies must reject tampered, stale, unsigned, placeholder, or out-of-scope profile material |
| SEC-ASSESS-PIN-CUSTODY | CVM and PED boundary | kernel receives, stores, logs, or derives clear PIN material instead of opaque PED handles | cvm::tests::offline_pin_requires_ped_owned_opaque_handle, cvm::tests::offline_pin_debug_redacts_ped_handle_values, trace::tests::replay_rejects_pin_verify_payload_custody, traceability_foundation::krn_pin_001_002_003_pinapi_001_002_cvmres_001_use_ped_owned_handles | PED integration review proving VERIFY and online PIN custody remain outside kernel memory and logs | kernel interfaces may pass only opaque PIN handles and status words, never clear PIN blocks or PIN digits |
| SEC-ASSESS-ODA-MATERIAL | ODA certificate recovery and cryptographic material | recovered public-key material, SDAD content, or issuer authentication data is logged or accepted without profile provenance | oda::tests::capk_lookup_requires_verified_integrity_and_unexpired_key, oda::tests::rejects_public_key_material_above_resource_limits, ffi::tests::runtime_cda_failure_sets_tvr_without_falling_back_to_dda, traceability_foundation::krn_sec_003_oda_002_capks_retain_signed_public_provenance | review of ODA error handling, material redaction, CAPK provenance, and CDA fail-closed behavior | ODA failures must set TVR/TSI evidence without exposing recovered material or silently weakening authentication method |
| SEC-ASSESS-ISSUER-SCRIPTS | issuer authentication and issuer script processing | issuer scripts execute across the wrong phase, hide failed critical commands, or leak script command payloads | issuer::tests::rejects_malformed_issuer_script_command_apdus, ffi::tests::critical_issuer_script_failure_stops_remaining_commands, ffi::tests::issuer_script_result_metadata_api_reports_phase_position_and_identifier, trace::tests::production_suppresses_issuer_script_command_data | negative tests for tag 71/tag 72 phase separation, critical command failure, retry status handling, and redacted result reporting | script execution order and result reporting must be phase-aware, bounded, redacted, and fail closed on critical command failure |
| SEC-ASSESS-REPORT-INTEGRITY | certification evidence and report workbench | repository-generated evidence is mistaken for external approval or attached to the wrong submitted build | reporting::tests::report_pack_json_lists_artifacts_reports_and_tools_without_approval_claims, freeze::tests::freeze_manifest_tracks_hash_slots_without_certification_claims, traceability_foundation::certification_freeze_manifest_is_reproducible_and_scoped, traceability_foundation::certification_report_workbench_is_reproducible_and_scoped | report-package review proving submitted-build hashes, finding disposition, and external approval attachments align | repository artifacts may support assessment but must not claim certification, approval, or closure of external gates |
